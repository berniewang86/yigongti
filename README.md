# 县域医共体一体化平台原型

这是用于领导汇报和早期需求确认的前端演示原型，围绕县人民医院牵头的紧密型县域医共体建设。

## 当前包含模块

- 管理端：县域运行驾驶舱、转诊监管、规则总览、转诊病种目录、转诊标准配置、慢病监管、医生协同监管、数据治理、急救随访监管、绩效医保、权限与规则。
- 管理端升级领导驾驶舱：核心目标、年度目标、责任部门、整改动作和重点督办。
- 新增病种规则：必须上转、会诊后上转、留基层管理、下转康复、资料清单和补偿约束。
- 新增转诊病种目录和转诊标准配置：展示病种分层、上转/留基层/下转规则、必带资料、绩效约束、启停状态和责任科室。
- 转诊监管和医生端我的转诊已嵌入四类上转路径：基层普通上转、急症绿色通道、会诊后上转、慢病异常上转。
- 四类上转路径支持选择后查看流程闭环、必带资料、绩效评分和协作服务动作。
- 关键卡片、任务、患者、报告、病种规则、机构绩效、协作服务和流程节点已接入业务详情抽屉，可查看关联证据、下一步动作和跨模块跳转。
- 管理端新增协作结算：服务项目配置、合规审核、结算台账、异常预警。
- 协作结算强化合规闭环：服务发生、规则校验、患者知情、县医院确认、质量审核、合规结算。
- 医生端：我的工作台、我的转诊、会诊协作、授权患者、随访任务、下沉带教、协作服务。
- 患者端：我的首页、我的转诊、预约就医、报告费用、家庭医生、慢病随访、知情授权。
- 三端权限：同一患者数据在管理端、医生端、患者端呈现不同粒度，操作边界不同。

## 本地预览

当前服务地址：

```text
http://localhost:5173/
```

如需重新启动，可在本目录运行：

```bash
python3 -m http.server 5173
```
